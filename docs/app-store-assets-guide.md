# App Store Assets Guide

## Required Assets for App Store Submission

### 1. App Icon

**Specifications:**
- **Size:** 1024x1024 pixels
- **Format:** PNG (no transparency)
- **Color Space:** sRGB or P3
- **File:** `assets/app-icon-1024.png`

**Design Guidelines:**
- Use GoldRock Health brand colors (Blue #4F46E5, Indigo gradient)
- Include medical cross or stethoscope symbol
- No transparency or alpha channels
- No rounded corners (iOS applies automatically)
- Should work well at small sizes (20x20 to 1024x1024)

**Icon Variations Needed:**
```
- 1024x1024 (App Store)
- 180x180 (iPhone Pro Max)
- 120x120 (iPhone standard)
- 76x76 (iPad)
- 152x152 (iPad Pro)
```

### 2. iPhone Screenshots

**Required Sizes:**

**6.7" Display (iPhone 15 Pro Max, 14 Pro Max)**
- Dimensions: 1290 x 2796 pixels
- Portrait orientation
- Minimum: 3 screenshots
- Maximum: 10 screenshots

**6.5" Display (iPhone 14 Plus, 13 Pro Max, etc.)**
- Dimensions: 1284 x 2778 pixels
- Portrait orientation
- Minimum: 3 screenshots

**5.5" Display (iPhone 8 Plus, older devices)**
- Dimensions: 1242 x 2208 pixels
- Portrait orientation  
- Minimum: 3 screenshots

### 3. iPad Screenshots (Optional but Recommended)

**12.9" Display (iPad Pro)**
- Dimensions: 2048 x 2732 pixels
- Portrait orientation
- Minimum: 3 screenshots

**11" Display (iPad Pro, iPad Air)**
- Dimensions: 1668 x 2388 pixels
- Portrait orientation
- Minimum: 3 screenshots

### 5. App Preview Video (Optional)

**Specifications:**
- **Duration:** 15-30 seconds
- **Format:** M4V, MP4, or MOV
- **Resolution:** Match screenshot sizes
- **Orientation:** Portrait
- **File Size:** Max 500 MB

**Video Script:**
1. Open app → Show medical disclaimer (2s)
2. Upload bill via camera (3s)
3. AI analysis in progress (2s)
4. Show analysis results with overcharges (5s)
5. View reduction strategies (3s)
6. Generate dispute letter (3s)
7. Show savings tracker (2s)
8. End screen: "Save thousands with GoldRock Health" (3s)

### 6. Additional Marketing Assets

**App Store Feature Graphic (if featured)**
- Size: 1920 x 1080 pixels
- Format: PNG or JPG
- Use: Featured placement in App Store

**Social Media Preview**
- Size: 1200 x 630 pixels
- Format: PNG or JPG
- Use: Link previews when sharing app

**Press Kit Images**
- Various sizes for media coverage
- Include logo variations (light/dark)
- Product screenshots in context

### 7. Asset Creation Checklist

**Pre-Production:**
- [ ] Define brand colors and typography
- [ ] Create app icon design mockups
- [ ] Plan screenshot flow and messaging
- [ ] Write compelling captions for each screenshot
- [ ] Prepare sample bills for screenshots

**Production:**
- [ ] Design 1024x1024 app icon (no transparency)
- [ ] Create all icon size variations
- [ ] Design screenshot templates with overlays
- [ ] Capture 6.7" iPhone screenshots (10 max)
- [ ] Capture 6.5" iPhone screenshots (10 max)
- [ ] Capture 5.5" iPhone screenshots (optional)
- [ ] Create iPad screenshots if supporting tablets
- [ ] Record app preview video (optional)

**Post-Production:**
- [ ] Optimize all images for file size
- [ ] Verify color space (sRGB or P3)
- [ ] Add localized text overlays
- [ ] Create dark mode variations if needed
- [ ] Test assets on actual devices

**Quality Assurance:**
- [ ] No pixelation or artifacts
- [ ] Text is legible at all sizes
- [ ] Colors match brand guidelines
- [ ] Screenshots show realistic use cases
- [ ] No placeholder or Lorem ipsum text
- [ ] All personal data is fictional/anonymized
- [ ] Medical disclaimer visible where needed

### 8. Screenshot Overlay Template

**Text Overlay Specifications:**
- **Font:** San Francisco Display (iOS system font) or Helvetica Neue
- **Title Size:** 60-80pt (bold)
- **Subtitle Size:** 32-44pt (regular)
- **Background:** Subtle gradient or semi-transparent overlay
- **Colors:** Brand blue (#4F46E5) with white text

**Example Overlay:**
```
[Top of screenshot]
─────────────────────
  AI Bill Analysis
  
  Find overcharges in
  under 60 seconds
─────────────────────
[Rest of screenshot shows app interface]
```

### 10. Tools for Asset Creation

**Icon Design:**
- Figma or Sketch (vector design)
- Adobe Illustrator
- Canva (templates available)

**Screenshots:**
- Figma Device Mockups
- Screenshot Creator apps
- Photoshop or Pixelmator Pro

**Video:**
- iPhone Screen Recording + iMovie
- Adobe Premiere Pro
- Final Cut Pro

**Optimization:**
- ImageOptim (reduce file size)
- TinyPNG (compression)
- Squoosh (format conversion)

### 11. Localization Considerations

If supporting multiple languages:
- [ ] Translate screenshot text overlays
- [ ] Adapt captions for cultural context
- [ ] Ensure text fits within overlay space
- [ ] Create language-specific app preview videos
- [ ] Translate app description and keywords

**Suggested Initial Languages:**
- English (US) - Primary
- Spanish (ES, MX) - Large US demographic
- Consider adding: Chinese, French, German

### 12. Final Submission Checklist

Before uploading to App Store Connect:
- [ ] All assets meet Apple's specifications
- [ ] No copyrighted content without permission
- [ ] Screenshots show current app version
- [ ] No references to other platforms (Android, web)
- [ ] Medical disclaimer visible in appropriate screenshots
- [ ] Pricing information is accurate
- [ ] No beta or "coming soon" features shown
- [ ] App name matches exactly across all assets
- [ ] Contact information is current

### 13. Asset Update Schedule

**When to Update Assets:**
- Major app redesign or rebrand
- Significant new features added
- Seasonal promotions or campaigns
- Competitor analysis shows better approaches
- User feedback suggests confusion
- Every 6-12 months for freshness

### 14. A/B Testing Recommendations

Test these variations:
- Screenshot order (which converts best first?)
- Text overlay vs. no overlay
- Feature focus (AI vs. savings vs. ease of use)
- Color schemes (trust-building blue vs. action-oriented green)
- Call-to-action wording

**Track:**
- App Store impressions
- Product page views
- Conversion rate
- Download velocity
