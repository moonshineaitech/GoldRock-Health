Project Title: MedTrainer AI - Interactive Medical Diagnosis Training Platform

Core Vision & Purpose
Create a professional-grade medical education platform that simulates real patient interactions for medical students, residents, and practicing physicians. The platform should feel like a premium medical education tool used by top medical schools, with the polish and user experience of a YCombinator-funded startup.

Technical Architecture Requirements
Frontend Stack:

React 18 + TypeScript + Vite
shadcn/ui component library with Tailwind CSS
Wouter for client-side routing
TanStack Query v5 for data fetching
Framer Motion for animations
React Hook Form with Zod validation
Backend Stack:

Express.js with TypeScript
PostgreSQL database with Drizzle ORM
RESTful API architecture
Session-based state management
External Integrations:

ElevenLabs API for text-to-speech voice synthesis
Web Speech API for voice input (browser-native)
Database Schema & Content
Medical Cases Table Structure:

id, name, age, difficulty (1-3), symptoms[], history[], responses{}, correctDiagnosis, specialty
Minimum 60+ cases across these specialties:
Cardiology, Neurology, Endocrinology, Gastroenterology, Oncology
Psychiatry, Infectious Disease, Dermatology, Orthopedics, Gynecology
Urology, ENT, Ophthalmology, Pulmonology, Hematology
Rheumatology, Nephrology, Emergency Medicine, Pediatrics
Progress Tracking:

User progress with timestamps, scores, completed cases
Diagnostic accuracy tracking
Achievement system for gamification
Core Features & User Journey
1. Landing Page (Professional Homepage):

Hero section with "MedTrainer AI" branding
Gradient backgrounds (slate-50 to indigo-100)
Feature highlights: Voice Interactions, AI-Powered Cases, Progress Tracking
Live statistics dashboard showing total cases available
Achievement cards with hover animations and glassmorphism effects
Professional CTA buttons leading to training
2. Training Case Selection Page (/training):

Grid layout of all available cases
Each case card shows: patient name, age, difficulty level, chief complaints preview
Search functionality by patient name, symptoms, or diagnosis
Filter by difficulty level (Foundation/Clinical/Expert)
Difficulty badges with color coding (green/amber/rose)
Professional card design with hover animations
3. Interactive Game Interface (/game/:id):

Patient Information Panel:
